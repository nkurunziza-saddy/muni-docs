# Installation

This page covers the installation process for Muni in detail.

## Prerequisites

Before installing Muni, ensure you have:

- **Node.js** (version 18 or higher)
- **pnpm** (recommended) or npm/yarn
- A code editor (VS Code recommended)

## Step-by-Step Installation

### 1. Create a New Project

```bash

# Create a new Next.js project

npx create-next-app@latest my-docs --typescript --tailwind --eslint --app

# Navigate to your project

cd my-docs
```

### 2. Install Dependencies

```bash

# Install required packages

pnpm add @next/mdx @mdx-js/loader @mdx-js/react
pnpm add remark-gfm remark-frontmatter remark-mdx-frontmatter
pnpm add rehype-slug rehype-autolink-headings

# Install UI components (optional)

pnpm add @radix-ui/react-navigation-menu lucide-react
```

### 3. Configure Next.js

Update your `next.config.mjs`:

```javascript
import createMDX from '@next/mdx'
import remarkGfm from 'remark-gfm'
import remarkFrontmatter from 'remark-frontmatter'
import remarkMdxFrontmatter from 'remark-mdx-frontmatter'
import rehypeSlug from 'rehype-slug'
import rehypeAutolinkHeadings from 'rehype-autolink-headings'

const withMDX = createMDX({
options: {
remarkPlugins: [remarkGfm, remarkFrontmatter, remarkMdxFrontmatter],
rehypePlugins: [rehypeSlug, rehypeAutolinkHeadings],
},
})

/*_ @type {import('next').NextConfig} _*/
const nextConfig = {
pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
}

export default withMDX(nextConfig)
```

### 4. Set Up Content Structure

Create the content directory:

```bash
mkdir -p content/pages
```

### 5. Start Development

```bash
pnpm dev
```

Your Muni documentation site should now be running at `http://localhost:3000`!

## Troubleshooting

### Common Issues

**MDX files not loading?**

- Check that your `next.config.mjs` includes the MDX configuration
- Ensure your content files have the `.mdx` extension
- Verify the file paths in your dynamic imports

**Styling issues?**

- Make sure Tailwind CSS is properly configured
- Check that your global styles are imported in `layout.tsx`

**Build errors?**

- Ensure all dependencies are installed
- Check for TypeScript errors in your components
- Verify that all MDX files have valid syntax
