# Getting Started

This guide will help you set up Muni and create your first documentation page.

## Prerequisites

- Node.js 18+
- pnpm (recommended) or npm

## Installation

1. **Clone the template**

   ```bash
   git clone https://github.com/nkurunziza-saddy/muni-docs
   cd muni-docs
   ```

2. **Install dependencies**

   ```bash
   pnpm install
   ```

3. **Start the development server**

   ```bash
   pnpm dev
   ```

4. **Open your browser**
   Navigate to `http://localhost:3000`

## Project Structure

```
muni-docs/
├── app/                    # Next.js app directory
│   ├── docs/              # Documentation pages
│   └── layout.tsx         # Root layout
├── components/            # React components
│   ├── ui/               # Base UI components
│   └── navigation-components/ # Navigation components
├── content/              # MDX content
│   └── pages/           # Documentation pages
├── styles/              # CSS files
│   ├── globals.css      # Global styles
│   └── color-themes.css # Theme definitions
├── muni.config.ts       # Configuration file
└── package.json
```

## Configuration

Edit `muni.config.ts` to customize your documentation:

```typescript
const muniConfig: MuniConfig = {
  title: "Your Documentation",
  version: "1.0.0",
  defaultTheme: "minimal",
  showFrontmatterMeta: true,
  headingLinks: [
    { title: "GitHub", href: "https://github.com/your-org/your-repo" },
  ],
  navigation: [
    { title: "Introduction", slug: "index" },
    { title: "Getting Started", slug: "getting-started" },
  ],
  githubRepo: "https://github.com/your-org/your-repo",
};
```

## Creating Pages

1. **Create a new MDX file** in `content/pages/`

   ```bash
   touch content/pages/my-page.mdx
   ```

2. **Add frontmatter** (optional)

   ```yaml
   ---
   title: "My Page"
   description: "A description of my page"
   author: "Your Name"
   date: "2024-01-01"
   ---
   ```

3. **Write your content** using Markdown

   ```markdown
   # My Page

   This is my documentation page.
   ```

4. **Add to navigation** in `muni.config.ts`
   ```typescript
   navigation: [{ title: "My Page", slug: "my-page" }];
   ```

## Writing Content

### Basic Markdown

Muni supports standard Markdown syntax:

```markdown
# Heading 1

## Heading 2

### Heading 3

**Bold text**
_Italic text_

- List item 1
- List item 2

1. Numbered item 1
2. Numbered item 2

[Link text](https://example.com)
```

### Code Blocks

Use triple backticks for code blocks:

````markdown
```javascript
function hello() {
  console.log("Hello, world!");
}
```
````

### Custom Components

You can use React components in your MDX files:

```jsx
<Button>Click me</Button>
```

## Themes

Muni comes with three themes:

- **minimal**: Clean white background
- **minimal-dark**: Dark theme
- **mono**: Monospace typography

Change the theme in `muni.config.ts`:

```typescript
defaultTheme: "minimal-dark";
```

## Building for Production

1. **Build the project**

   ```bash
   pnpm build
   ```

2. **Start the production server**
   ```bash
   pnpm start
   ```

## Troubleshooting

### Common Issues

**Build fails with MDX errors**
```bash
# Clear cache and reinstall
rm -rf .next node_modules
pnpm install
pnpm build
```

**Theme not applying correctly**
- Check that your `muni.config.ts` has a valid theme name
- Ensure CSS files are properly imported
- Clear browser cache

**Search not working**
- Run `pnpm run generate-search` to regenerate search data
- Check that `public/search-data.json` exists
- Verify content files have proper frontmatter

**Styling issues**
- Ensure Tailwind CSS is properly configured
- Check for conflicting CSS rules
- Verify all required CSS files are imported

### Performance Tips

1. **Optimize images**: Use Next.js Image component
2. **Minimize bundle size**: Remove unused dependencies
3. **Enable compression**: Configure your hosting platform
4. **Use static generation**: All pages are pre-rendered for speed

### Development Workflow

1. **Make changes** to content or components
2. **Test locally** with `pnpm dev`
3. **Build and test** with `pnpm build && pnpm start`
4. **Deploy** to your hosting platform
