# Configuration

Configuration for a Muni project is handled in two main places: `muni.config.ts` for site-specific settings like navigation, and `next.config.ts` for the underlying MDX and Next.js behavior.

## `muni.config.ts`

This is the primary configuration file for your site's content and navigation. It exports a `muniConfig` object.

### `MuniConfig` Interface

```typescript
interface MuniConfig {
  title: string;
  navigation: NavigationItem[];
}

interface NavigationItem {
  title: string;
  slug: string;
  items?: NavigationItem[];
}
```

### `title`

-   **Type:** `string`
-   **Description:** The main title of your documentation site. It's used in the browser tab, metadata, and can be used in your site header.

### `navigation`

-   **Type:** `NavigationItem[]`
-   **Description:** An array of objects that defines the structure of your sidebar navigation.

Each `NavigationItem` has the following properties:

-   `title` (string): The text displayed in the navigation sidebar.
-   `slug` (string): The URL slug for the page. This must match the file path in `content/pages/` (without the `.mdx` extension). For the homepage, use `index`.
-   `items` (optional `NavigationItem[]`): An optional array of nested navigation items to create collapsible sections in the sidebar.

### Example Configuration

```ts [muni.config.ts]
export const muniConfig = {
  title: "My Awesome Project",
  navigation: [
    {
      title: "Introduction",
      slug: "index",
    },
    {
      title: "Getting Started",
      slug: "getting-started",
    },
    {
      title: "Core Concepts",
      slug: "guide/index",
      items: [
        { title: "Writing Content", slug: "guide/writing-content" },
        { title: "Code Blocks", slug: "guide/code-blocks" },
      ],
    },
  ],
};
```

## `next.config.ts`

This file controls the Next.js build process, including how MDX files are handled. Muni comes with a pre-configured set of Remark and Rehype plugins to add advanced features to your Markdown.

### Default Plugins

Muni's default configuration includes plugins for:

-   **GFM (GitHub Flavored Markdown):** Adds support for tables, strikethroughs, etc.
-   **Frontmatter:** Parses the YAML frontmatter from your files.
-   **Code Block Titles:** Adds titles to code blocks.
-   **Callouts:** Enables `:::` directives for callouts.
-   **Steps:** Enables the `:::steps` component.
-   **Slug & Autolink Headings:** Automatically adds `id`s to headings and links them.

### Customizing Plugins

You can add your own Remark or Rehype plugins by modifying the `options` object in `next.config.ts`.

```ts [next.config.ts]
import createMDX from '@next/mdx';
import remarkToc from 'remark-toc'; // Example: adding a TOC plugin

const withMDX = createMDX({
  options: {
    remarkPlugins: [
      // ... existing Muni plugins
      remarkToc, // Add your new plugin here
    ],
    rehypePlugins: [
      // ... existing Muni plugins
    ],
  },
});

// ... rest of the config

export default withMDX(nextConfig);
```
