# Markdown Features

Muni supports standard CommonMark, GitHub Flavored Markdown (GFM), and custom MDX components.

## Basic Syntax

Standard markdown syntax is supported for all basic text formatting.

### Text Formatting

- **Bold text** using `**text**`
- _Italic text_ using `_text_` or `*text*`
- ~~Strikethrough~~ using `~~text~~`
- `Inline code` using backticks

### Lists

#### Unordered Lists
- Item 1
- Item 2
  - Sub-item 2.1
  - Sub-item 2.2

#### Ordered Lists
1. First step
2. Second step
   1. Sub-step 2.1
   2. Sub-step 2.2

## Technical Features

### Code Blocks

Code blocks are powered by Shiki with support for multiple themes, line numbering, and rich annotations.

#### Basic Code Block

```typescript
const hello = "world";
console.log(hello);
```

#### Code Block with Title

```typescript title="main.ts"
function main() {
  console.log("Muni Docs Engine");
}
```

#### Line Numbers

```typescript showLineNumbers
// This block shows line numbers
const list = [1, 2, 3];
list.forEach(item => {
  console.log(item * 2);
});
```

#### Annotations

##### Highlighting Lines

```typescript {1,3}
// Line 1 is highlighted via meta string
const x = 10;
// Line 3 is also highlighted
```

```typescript
// [!code hl]
console.log("This line is highlighted via comment");
console.log("This one is not");
```

##### Diff Markers

```typescript
function update() {
  const old = false; // [!code --]
  const fresh = true; // [!code ++]
}
```

##### Focus

```typescript
function complex() {
  console.log("Irrelevant");
  console.log("IMPORTANT"); // [!code focus]
  console.log("Irrelevant");
}
```

## Best Practices

1. **Keep it simple** - Use standard markdown whenever possible.
2. **Use components sparingly** - Components are great for emphasis but can clutter the reading experience.
3. **Organize with headings** - Proper heading hierarchy helps both readers and the search engine.
4. **Annotate code** - Use Shiki annotations to call out specific parts of your code.
